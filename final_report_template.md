## 2. 專題功能與設計規格說明

### 2.1 專題背景與目標
本專案為一個終端機互動式商店 POS (Point of Sale) 與庫存管理系統。旨在提供商家一個直覺、防呆且具備動態定價（如過期促銷、季末折扣、保固增費）的結帳收銀與庫存異動工具。

### 2.2 物件導向三大特性之應用
1. **封裝 (Encapsulation)**：商品的基本屬性（ID、價格、庫存等）皆在 `Product` 類別中聲明為受保護的成員（`protected`），並透過公開的 Getters 與 Setters 進行安全存取。
2. **繼承 (Inheritance)**：定義了抽象基底類別 `Product`，並派生出三個子類別：
   * `FoodProduct`（食品類）：增加「有效天數」屬性。
   * `ClothingProduct`（服飾類）：增加「尺寸」與「材質」屬性。
   * `ElectronicsProduct`（電子類）：增加「保固月數」屬性。
3. **多型 (Polymorphism)**：
   * 透過在基底類別定義純虛擬函式 `virtual double getDiscountedPrice() const = 0` 與 `virtual std::string getProductType() const = 0`。
   * 不同的衍生類別各自 override 這些函式，實現執行期的動態繫結（Dynamic Binding）。例如：購物車在結帳時，只需遍歷 `Product*`，即可自動根據該商品的實際衍生類型計算出對應的折扣後售價。

### 2.3 檔案讀檔與寫檔功能
* **載入**：系統啟動時讀取同目錄下的 `inventory.txt`，分析每一行的 CSV 字串，根據開頭的商品類型標籤（Food / Clothing / Electronics），動態以 `std::make_unique` 實例化對應的衍生類別並加入記憶體。
* **寫入**：系統提供手動存檔與結束程式存檔選項，會呼叫每個商品的虛擬函式 `serialize()`，將目前的狀態（包含修改後的庫存量）以 CSV 格式覆寫回 `inventory.txt`。

### 2.4 STL (標準樣板函式庫) 之使用
* `std::vector`：作為商品資料庫的主容器。
* `std::unique_ptr`：用於包裝衍生類別指標，實現安全的記憶體資源自動管理（RAII），防止記憶體洩漏。
* `std::sort` 與 Lambda：用於實作商品的多條件排序，讓商家能自由按 ID 升序、折實售價升序或庫存量降序查看。
* `std::stringstream`：用於檔案每行資料的逗號解析與字串轉換。

## 3. 開發迭代流程與進度

本專案採用敏捷的迭代開發模式，分為四個階段完成：
* **迭代一 (核心多型結構)**：完成 `Product.h` 基底類別與 `FoodProduct.h`、`ClothingProduct.h`、`ElectronicsProduct.h` 衍生類別的設計，並實作各品項特有的定價折扣演算法。
* **迭代二 (資料庫存管理器與檔案 I/O)**：完成 `InventoryManager` 類別，實作以 `std::vector<std::unique_ptr<Product>>` 管理商品，並撰寫 CSV 格式的檔案讀寫解析。
* **迭代三 (終端機選單與收銀模擬)**：在 `main.cpp` 中建立完整主控制迴圈，加入購物車 POS 結帳邏輯，以及針對使用者輸入的防呆流控制。
* **迭代四 (文檔與驗證)**：建立 `inventory.txt` 預設測試資料，撰寫說明文件，並手動測試各功能。
